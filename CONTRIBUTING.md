# Contributing to Synology ActiveBackup API Documentation

Thank you for your interest in contributing! This documentation is community-driven and we welcome all contributions.

## How to Contribute

### Reporting Issues

Found an error or have a suggestion?

1. Check if the issue already exists in [GitHub Issues](https://github.com/milanese-org/synology-activebackup-api/issues)
2. If not, create a new issue with:
   - Clear title and description
   - Steps to reproduce (if applicable)
   - Expected vs actual behavior
   - Your environment (NAS model, DSM version, ActiveBackup version)

### Improving Documentation

Documentation improvements are always welcome:

1. Fork the repository
2. Create a new branch (`git checkout -b improve-docs`)
3. Make your changes
4. Test your changes locally
5. Commit with clear message (`git commit -m "Improve authentication docs"`)
6. Push to your fork (`git push origin improve-docs`)
7. Create a Pull Request

### Adding Examples

We love code examples! To add an example:

1. Choose the appropriate language folder in `docs/examples/`
2. Create a well-commented example file
3. Test the example against a real ActiveBackup instance
4. Add a README.md explaining what the example does
5. Submit a Pull Request

### Testing New API Methods

Discovered a new API method or parameter?

1. Test it thoroughly on your system
2. Document the request and response
3. Add it to the appropriate API reference file
4. Include any error cases you encountered
5. Submit a Pull Request with your findings

## Documentation Standards

### Markdown Style

- Use ATX-style headers (`#` not `===`)
- Include code blocks with language specification
- Use tables for structured data
- Add links to related documentation

### Code Examples

- Include complete, runnable examples
- Add comments explaining key steps
- Handle errors gracefully
- Use realistic variable names
- Test before submitting

### API Documentation

Each API method should include:

- Method name and description
- HTTP method (GET/POST)
- Required and optional parameters
- Example request
- Example response
- Error codes
- Notes about special behavior

## Code of Conduct

### Our Standards

- Be respectful and inclusive
- Welcome newcomers
- Accept constructive criticism
- Focus on what's best for the community
- Show empathy towards others

### Unacceptable Behavior

- Harassment or discriminatory language
- Trolling or insulting comments
- Personal or political attacks
- Publishing others' private information
- Other unprofessional conduct

## Questions?

- Open a [GitHub Discussion](https://github.com/milanese-org/synology-activebackup-api/discussions)
- Check existing documentation
- Review closed issues for similar questions
